Portrait theme
Toggle the Portrait field into Excalidraw — first click lazy-loads the editor bundle. Toggle the Origin Story field into the markdown editor.
What's happening: Stock inputs (Text, Number, Option, TextArea) sit next to two complex InputTypes — Diagram (Portrait) and RichText (Origin Story) — in the same form, all driven by the same marshalDataFromAppDataToView pipeline. The portrait SVGs use var(--diagram-ink, …) with hex fallbacks, so swapping the theme swatches recolors them live without re-rendering. Loading a new hero from the dropdown folds any open editors back into view mode first, so the swap doesn't strand CodeMirror or Excalidraw on yesterday's character.
